JwtUtil.java
JAVA
@Component
public class JwtUtil {

    @Value("${jwt.secret}")
    private String secretKey;

    @Value("${jwt.access-token-expiration}")
    private long accessExpiry;  // 15 min in ms

    @Value("${jwt.refresh-token-expiration}")
    private long refreshExpiry; // 7 days in ms

    // ① application.properties-ල ඇති secret string → HMAC-SHA256 Key
    private SecretKey getSigningKey() {
        return Keys.hmacShaKeyFor(secretKey.getBytes(StandardCharsets.UTF_8));
    }

    // ② Access Token — userId + role payload-ල දාලා JWT create
    public String generateAccessToken(String userId, String role) {
        return Jwts.builder()
                .subject(userId)
                .claim("role", role)        // Admin, Cashier etc.
                .issuedAt(new Date())
                .expiration(new Date(System.currentTimeMillis() + accessExpiry))
                .signWith(getSigningKey())  // Secret Key-ල sign!
                .compact();               // → "xxxxx.yyyyy.zzzzz"
    }

    // ③ Refresh Token — userId only, longer expiry
    public String generateRefreshToken(String userId) {
        return Jwts.builder()
                .subject(userId)
                .issuedAt(new Date())
                .expiration(new Date(System.currentTimeMillis() + refreshExpiry))
                .signWith(getSigningKey())
                .compact();
    }

    // ④ Token valid ද? Signature OK? Expired නෑ? — Database query නෑ!
    public boolean isTokenValid(String token) {
        try {
            Jwts.parser().verifyWith(getSigningKey()).build()
               .parseSignedClaims(token);
            return true;
        } catch (JwtException e) { return false; }
    }

    // ⑤ Token-ල userId extract
    public String extractUserId(String token) {
        return Jwts.parser().verifyWith(getSigningKey()).build()
                .parseSignedClaims(token).getPayload().getSubject();
    }

    // ⑥ Token-ල role extract
    public String extractRole(String token) {
        return Jwts.parser().verifyWith(getSigningKey()).build()
                .parseSignedClaims(token).getPayload()
                .get("role", String.class);
    }
}
JwtUtil.java — සම්පූර්ණ explanation:
  • getSigningKey() application.properties Secret String-ල HMAC-SHA256 cryptographic key-ල convert කරනවා. Token sign කිරීමටත් verify කිරීමටත් same key use. .env-ල store කිරීම = production best practice.
  • generateAccessToken() userId + role payload-ල දාලා, 15 min expiry set කරලා, Secret Key-ල sign කරලා "xxxxx.yyyyy.zzzzz" JWT string return කරනවා. Passwords, Credit Card numbers payload-ල දාන්නේ නෑ — Base64 decode කරන්නට ඕනෑ කෙනෙකුට හැකිය!
  • generateRefreshToken() Access Token expire වෙද්දී නව Access Token ගන්නට use කරන Refresh Token. Expiry = 7 days. HttpOnly Cookie-ල store.
  • isTokenValid() Database query නෑ! JJWT library-ල Secret Key-ල signature verify + expiry check. Tampered token, expired token = JwtException → false return.
  • extractUserId() / extractRole() Valid token-ල Payload-ල userId සහ role read කරනවා. Spring Security context-ල set කිරීමට use.

@RestController
@RequestMapping("/api/auth")
public class AuthController {

    private final UserService userService;
    private final JwtUtil jwtUtil;

    // ① POST /api/auth/login — Cashier credentials receive
    @PostMapping("/login")
    public ResponseEntity<AuthResponse> login(@RequestBody LoginRequest req) {

        // ② Database-ල credentials validate — Login-ලදී ONLY DB query!
        User user = userService.authenticate(req.getUsername(), req.getPassword());

        if (user == null) {
            return ResponseEntity.status(HttpStatus.UNAUTHORIZED).build();
        }

        // ③ Valid නම් — Access Token (15min) + Refresh Token (7days) generate
        String accessToken  = jwtUtil.generateAccessToken(user.getId(), user.getRole());
        String refreshToken = jwtUtil.generateRefreshToken(user.getId());

        // ④ Refresh Token — HttpOnly Secure Cookie-ල (JavaScript-ල access කරන්නට බෑ!)
        ResponseCookie cookie = ResponseCookie.from("refreshToken", refreshToken)
                .httpOnly(true)          // XSS protection — JS access blocked
                .secure(true)            // HTTPS only
                .sameSite("Strict")      // CSRF protection
                .path("/")
                .maxAge(Duration.ofDays(7))
                .build();

        // ⑤ Access Token — Response body-ල (Frontend memory-ල store කරයි)
        return ResponseEntity.ok()
                .header(HttpHeaders.SET_COOKIE, cookie.toString())
                .body(new AuthResponse(accessToken));
    }

    // ⑥ POST /api/auth/refresh — Refresh Token-ල Access Token renew
    @PostMapping("/refresh")
    public ResponseEntity<AuthResponse> refresh(@CookieValue("refreshToken") String refreshToken) {
        if (!jwtUtil.isTokenValid(refreshToken)) {
            return ResponseEntity.status(HttpStatus.UNAUTHORIZED).build();
        }
        String userId = jwtUtil.extractUserId(refreshToken);
        User user = userService.findById(userId);
        String newAccessToken = jwtUtil.generateAccessToken(userId, user.getRole());
        return ResponseEntity.ok(new AuthResponse(newAccessToken));
    }
}

Step 4 — JWT Filter (සෑම Request-ලාම Intercept + Verify)

JwtAuthFilter.java
JAVA
@Component
public class JwtAuthFilter extends OncePerRequestFilter {

    private final JwtUtil jwtUtil;

    @Override
    protected void doFilterInternal(
            HttpServletRequest request,
            HttpServletResponse response,
            FilterChain chain) throws IOException, ServletException {

        // ① "Authorization: Bearer xxxxx.yyyyy.zzzzz" header extract
        String authHeader = request.getHeader("Authorization");

        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            chain.doFilter(request, response);
            return; // Token නෑ නම් skip — login page etc.
        }

        // ② "Bearer " (7 chars) remove → pure JWT token string
        String token = authHeader.substring(7);

        // ③ Signature verify + expiry check — NO DATABASE CALL!
        if (jwtUtil.isTokenValid(token)) {
            String userId = jwtUtil.extractUserId(token);
            String role   = jwtUtil.extractRole(token);

            // ④ Spring Security Context-ල authenticated user set
            UsernamePasswordAuthenticationToken auth =
                new UsernamePasswordAuthenticationToken(
                    userId, null,
                    List.of(new SimpleGrantedAuthority("ROLE_" + role))
                );
            SecurityContextHolder.getContext().setAuthentication(auth);
        }

        chain.doFilter(request, response);
    }
}
SecurityConfig.java
JAVA
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    private final JwtAuthFilter jwtAuthFilter;

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        return http
            .csrf(csrf -> csrf.disable())
            .sessionManagement(sm -> sm
                // JWT = Stateless. Server session create නොකරයි!
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/**").permitAll()  // Login = free
                .requestMatchers("/api/admin/**").hasRole("ADMIN")
                .requestMatchers("/api/bills/**").hasAnyRole("ADMIN", "CASHIER")
                .anyRequest().authenticated()
            )
            // JWT Filter — UsernamePasswordAuthenticationFilter-ලට BEFORE
            .addFilterBefore(jwtAuthFilter,
                UsernamePasswordAuthenticationFilter.class)
            .build();
    }
}
JWT Filter — සෑම API request-ලාම මෙය run වෙනවා:
  • OncePerRequestFilter Controller-ලා reach වෙන්නට කලිනම JwtAuthFilter run. Authorization header check කරනවා. Login/register endpoints token නැතිව skip.
  • substring(7) Header value = "Bearer eyJ...". "Bearer " = 7 characters. substring(7) = pure JWT token.
  • isTokenValid() Database query සිදු නොවේ! JJWT library Secret Key-ල Signature verify + expiry check. ඕනෑ Server-ලා (1, 2, 3) verify කිරීමට හැකිය — Stateless!
  • SecurityContextHolder Valid token නම් userId + role Spring Security Context-ල set. @PreAuthorize("hasRole('ADMIN')") annotations ඊට rely කරනවා.
  • STATELESS SecurityConfig-ල SessionCreationPolicy.STATELESS = Server-side session කිසිවිටකත් create නොකිරීමට instruct. JWT-ල essence.

// ① Protected API call — Authorization header-ල Access Token
const getBills = async (accessToken) => {
    const response = await fetch('/api/bills', {
        headers: {
            'Authorization': `Bearer ${accessToken}`,  // JWT!
            'Content-Type': 'application/json'
        },
        credentials: 'include'  // Cookie auto-include
    });

    // ② Access Token expire (401) → Refresh Token-ල renew
    if (response.status === 401) {
        const newToken = await refreshAccessToken();
        return getBills(newToken); // Retry
    }

    return response.json();
};

// ③ HttpOnly Cookie-ල Refresh Token auto-sent → new Access Token
const refreshAccessToken = async () => {
    const res = await fetch('/api/auth/refresh', {
        method: 'POST',
        credentials: 'include'  // HttpOnly Cookie auto-sent!
    });
    const data = await res.json();
    setAccessToken(data.accessToken);
    return data.accessToken;
};

06 — Session-Based vs JWT
✗ Session-Based (Old)
  • Server memory/DB-ල session store
  • Server 1-ල login → Server 2-ල fail
  • Load balancing = Redis extra
  • Mobile apps-ල cookies fail
  • CORS cross-origin issues
  • හැම request-ලාම DB session lookup
✓ JWT / Token-Based (Modern)
  • Server-side state නෑ — Stateless
  • ඕනෑ Server verify කරනවා — Scale freely

07 — Best Practices
01
🚫 Sensitive Data එපා
Payload Base64 decode කරන්නට හැකිය. Passwords, Card numbers JWT-ල නෑ. User ID, Role only.
02
🔑 Strong Secret Key
256-bit+ random. Hardcode නෑ. .env file. Production-ල rotate. application.properties-ල @Value inject.
03
⏰ Short Access Token
15 min expiry. Stolen token damage window කෙටිය. Refresh Token-ල seamlessly renew.
04
🍪 HttpOnly Cookie
Refresh Token HttpOnly Secure Cookie-ල. JS access blocked = XSS protection. SameSite=Strict = CSRF protection.
05
📚 Library Use
io.jsonwebtoken (Spring) / jsonwebtoken (Node). Manual JWT implementation = vulnerabilities risk. Library use = safe.
06
🔄 Token Rotation
Refresh Token use-ලා නව Refresh Token issue කරලා old invalidate. Replay attacks prevent. DB-ල revocation list.
