JwtUtil.java
JAVA
@Component
public class JwtUtil {

    @Value("${jwt.secret}")
    private String secretKey;

    @Value("${jwt.access-token-expiration}")
    private long accessExpiry;  // 15 min in ms

    @Value("${jwt.refresh-token-expiration}")
    private long refreshExpiry; // 7 days in ms

    // ① application.properties හි ඇති secret string → HMAC-SHA256 Key
    private SecretKey getSigningKey() {
        return Keys.hmacShaKeyFor(secretKey.getBytes(StandardCharsets.UTF_8));
    }

    // ② Access Token — userId සහ role payload එකට දාලා JWT create කිරීම
    public String generateAccessToken(String userId, String role) {
        return Jwts.builder()
                .subject(userId)
                .claim("role", role)        // Admin, Cashier etc.
                .issuedAt(new Date())
                .expiration(new Date(System.currentTimeMillis() + accessExpiry))
                .signWith(getSigningKey())  // Secret Key එකෙන් sign!
                .compact();               // → "xxxxx.yyyyy.zzzzz"
    }

    // ③ Refresh Token — userId පමණි, longer expiry
    public String generateRefreshToken(String userId) {
        return Jwts.builder()
                .subject(userId)
                .issuedAt(new Date())
                .expiration(new Date(System.currentTimeMillis() + refreshExpiry))
                .signWith(getSigningKey())
                .compact();
    }

    // ④ Token valid ද? Signature OK ද? Expired නැද්ද? — Database query නෑ!
    public boolean isTokenValid(String token) {
        try {
            Jwts.parser().verifyWith(getSigningKey()).build()
               .parseSignedClaims(token);
            return true;
        } catch (JwtException e) { return false; }
    }

    // ⑤ Token එකෙන් userId extract කිරීම
    public String extractUserId(String token) {
        return Jwts.parser().verifyWith(getSigningKey()).build()
                .parseSignedClaims(token).getPayload().getSubject();
    }

    // ⑥ Token එකෙන් role extract කිරීම
    public String extractRole(String token) {
        return Jwts.parser().verifyWith(getSigningKey()).build()
                .parseSignedClaims(token).getPayload()
                .get("role", String.class);
    }
}
JwtUtil.java — සම්පූර්ණ පැහැදිලි කිරීම:
  • getSigningKey() මගින් application.properties හි ඇති Secret String එක HMAC-SHA256 cryptographic key එකක් බවට convert කරනවා. Token එක sign කිරීමටත් verify කිරීමටත් භාවිතා කරන්නේ එකම key එකයි (same key). මෙය .env file එකක store කිරීම production best practice එකක් වෙනවා.
  • generateAccessToken() මගින් userId සහ role එක payload එකට දාලා, 15 min expiry එකක් set කරලා, Secret Key එකෙන් sign කරලා "xxxxx.yyyyy.zzzzz" ආකාරයේ JWT string එකක් return කරනවා. Passwords, Credit Card numbers වගේ දේවල් කිසිම විටෙක payload එකට දාන්නේ නෑ — මන්ද Base64 decode කරන්න ඕනෑම කෙනෙකුට පුළුවන්!
  • generateRefreshToken() මගින් Access Token එක expire වෙද්දී අලුත් Access Token එකක් ලබාගන්න පාවිච්චි කරන Refresh Token එක හදනවා. මෙහි Expiry එක දවස් 7යි. මෙය HttpOnly Cookie එකක store කරනවා.
  • isTokenValid() මෙහිදී Database query එකක් run වෙන්නේ නෑ! JJWT library එක මගින් Secret Key එක භාවිතා කරලා signature එක verify කරනවා වගේම expiry එක check කරනවා. වෙනස් කළ (tampered) token හෝ expired token එකකදී JwtException එකක් හරහා false return වෙනවා.
  • extractUserId() / extractRole() මගින් Valid token එකක Payload එකෙන් userId සහ role එක read කරනවා. මේවා Spring Security context එක set කරන්න පාවිච්චි කරනවා.

@RestController
@RequestMapping("/api/auth")
public class AuthController {

    private final UserService userService;
    private final JwtUtil jwtUtil;

    // ① POST /api/auth/login — Cashier credentials receive
    @PostMapping("/login")
    public ResponseEntity<AuthResponse> login(@RequestBody LoginRequest req) {

        // ② Database එකෙන් credentials validate කිරීම — Login වීමේදී ONLY DB query!
        User user = userService.authenticate(req.getUsername(), req.getPassword());

        if (user == null) {
            return ResponseEntity.status(HttpStatus.UNAUTHORIZED).build();
        }

        // ③ Valid නම් — Access Token (15min) + Refresh Token (7days) generate
        String accessToken  = jwtUtil.generateAccessToken(user.getId(), user.getRole());
        String refreshToken = jwtUtil.generateRefreshToken(user.getId());

        // ④ Refresh Token — HttpOnly Secure Cookie එකකට (JavaScript හරහා access කරන්නට බෑ!)
        ResponseCookie cookie = ResponseCookie.from("refreshToken", refreshToken)
                .httpOnly(true)          // XSS protection — JS access blocked
                .secure(true)            // HTTPS only
                .sameSite("Strict")      // CSRF protection
                .path("/")
                .maxAge(Duration.ofDays(7))
                .build();

        // ⑤ Access Token — Response body එකට (Frontend memory එකේ store කරයි)
        return ResponseEntity.ok()
                .header(HttpHeaders.SET_COOKIE, cookie.toString())
                .body(new AuthResponse(accessToken));
    }

    // ⑥ POST /api/auth/refresh — Refresh Token එකෙන් Access Token renew කිරීම
    @PostMapping("/refresh")
    public ResponseEntity<AuthResponse> refresh(@CookieValue("refreshToken") String refreshToken) {
        if (!jwtUtil.isTokenValid(refreshToken)) {
            return ResponseEntity.status(HttpStatus.UNAUTHORIZED).build();
        }
        String userId = jwtUtil.extractUserId(refreshToken);
        User user = userService.findById(userId);
        String newAccessToken = jwtUtil.generateAccessToken(userId, user.getRole());
        return ResponseEntity.ok(new AuthResponse(newAccessToken));
    }
}

Login Controller — මෙයින් කරන්නේ:
  • @PostMapping("/login") මගින් Client POST කරන username සහ password ලබාගන්නවා. Database query එකක් සිදුවෙන්නේ මෙහිදී පමණයි — userService.authenticate() මගින් DB එකේ ඇති credentials check කරනවා.
  • generateAccessToken() Valid user කෙනෙක් නම් Access Token එකක් (15 min) සහ Refresh Token එකක් (7 days) generate කරනවා. ඒ සඳහා JwtUtil.java හි define කරපු methods call කරනවා.

  • sameSite("Strict") ලබාදීමෙන් CSRF attacks වලින් ආරක්ෂාව ලැබෙනවා. Cross-site requests වලදී cookie එක automatic send වෙන්නේ නෑ.
  • /refresh endpoint එක හරහා Access Token එක expire වුණාම Refresh Token එක භාවිතා කරලා user ට නැවත login වෙන්න (re-login) නොදී නව Access Token එකක් issue කරනවා. මෙහිදී Cookie එක browser එක මගින් auto-send වෙනවා.

@Component
public class JwtAuthFilter extends OncePerRequestFilter {

    private final JwtUtil jwtUtil;

    @Override
    protected void doFilterInternal(
            HttpServletRequest request,
            HttpServletResponse response,
            FilterChain chain) throws IOException, ServletException {

        // ① "Authorization: Bearer xxxxx.yyyyy.zzzzz" header extract
        String authHeader = request.getHeader("Authorization");

        if (authHeader == null || !authHeader.startsWith("Bearer ")) {
            chain.doFilter(request, response);
            return; // Token නෑ නම් skip — login page etc.
        }

        // ② "Bearer " (7 chars) remove → pure JWT token string
        String token = authHeader.substring(7);

        // ③ Signature verify + expiry check — NO DATABASE CALL!
        if (jwtUtil.isTokenValid(token)) {
            String userId = jwtUtil.extractUserId(token);
            String role   = jwtUtil.extractRole(token);

            // ④ Spring Security Context එකේ authenticated user set කිරීම
            UsernamePasswordAuthenticationToken auth =
                new UsernamePasswordAuthenticationToken(
                    userId, null,
                    List.of(new SimpleGrantedAuthority("ROLE_" + role))
                );
            SecurityContextHolder.getContext().setAuthentication(auth);
        }

        chain.doFilter(request, response);
    }
}
SecurityConfig.java
JAVA
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    private final JwtAuthFilter jwtAuthFilter;

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        return http
            .csrf(csrf -> csrf.disable())
            .sessionManagement(sm -> sm
                // JWT = Stateless. Server session create නොකරයි!
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/**").permitAll()  // Login = free
                .requestMatchers("/api/admin/**").hasRole("ADMIN")
                .requestMatchers("/api/bills/**").hasAnyRole("ADMIN", "CASHIER")
                .anyRequest().authenticated()
            )
            // JWT Filter — UsernamePasswordAuthenticationFilter එකට BEFORE
            .addFilterBefore(jwtAuthFilter,
                UsernamePasswordAuthenticationFilter.class)
            .build();
    }
}
JWT Filter — සෑම API request එකකදීම මෙය run වෙනවා:
  • OncePerRequestFilter Controller එකට reach වෙන්න කලින්ම මෙය හරහා JwtAuthFilter එක run වෙනවා. මෙයින් Authorization header එක check කරනවා. Login/register endpoints වලදී token එකක් නැති නිසා මෙය skip කරනවා.
  • substring(7) මගින් "Bearer eyJ..." කියන Header value එකේ මුල් characters 7 ("Bearer ") අයින් කරලා pure JWT token එක ලබාගන්නවා.
  • isTokenValid() මෙහිදී Database query සිදු නොවේ! JJWT library එක Secret Key එක මගින් Signature එක verify කරලා expiry check කරනවා. ඕනෑම Server එකකදී (1, 2, 3) මෙය verify කරන්න පුළුවන් — මන්ද මෙය Stateless නිසා!
  • SecurityContextHolder මගින් Valid token එකකදී userId සහ role එක Spring Security Context එකට set කරනවා. @PreAuthorize("hasRole('ADMIN')") වැනි annotations මේ මත රඳා පවතිනවා.
  • STATELESS SecurityConfig හි SessionCreationPolicy.STATELESS ලෙස යෙදීමෙන්, Server-side session කිසිවිටකත් create නොකරන ලෙස instruct කරනවා. මෙය තමයි JWT හි මූලිකම හරය.

// ① Protected API call — Authorization header එකේ Access Token එක යැවීම
const getBills = async (accessToken) => {
    const response = await fetch('/api/bills', {
        headers: {
            'Authorization': `Bearer ${accessToken}`,  // JWT!
            'Content-Type': 'application/json'
        },
        credentials: 'include'  // Cookie auto-include
    });

    // ② Access Token expire (401) → Refresh Token එකෙන් renew කිරීම
    if (response.status === 401) {
        const newToken = await refreshAccessToken();
        return getBills(newToken); // Retry
    }

    return response.json();
};

// ③ HttpOnly Cookie එකේ ඇති Refresh Token එක auto-sent → new Access Token
const refreshAccessToken = async () => {
    const res = await fetch('/api/auth/refresh', {
        method: 'POST',
        credentials: 'include'  // HttpOnly Cookie auto-sent!
    });
    const data = await res.json();
    setAccessToken(data.accessToken);
    return data.accessToken;
};

Frontend — Token use කරන ආකාරය:
  • setAccessToken() මගින් Access Token එක React useState (memory) එකේ store කරනවා. Page එක refresh කළාම මෙය clear වෙනවා — එය හිතාමතාම කරන design එකක් (design by intent). localStorage භාවිතා නොකිරීම නිසා XSS attacks වලදී JavaScript හරහා token එක read කරන එක වළක්වනවා.
  • credentials: 'include' යෙදීමෙන් HttpOnly Cookie එක (Refresh Token) automatically fetch request එකට include කරන්න කියලා browser එකට instruct කරනවා. මෙහිදී manual cookie handling අවශ්‍ය වෙන්නේ නෑ.
  • Authorization: Bearer හැම protected API request එකකම header එකේ Authorization: Bearer ලෙස JWT එක යවනවා. JwtAuthFilter හරහා server side එකෙන් මෙය intercept කරලා verify කරනවා. Database query කිසිවක් නෑ!
  • 401 → refresh flow යනු Access Token එක expire වුණාම ලැබෙන 401 response එකයි. ඉන්පසු Refresh Token එක මගින් /api/auth/refresh call කරනවා. එවිට User ට නැවත re-login නොවී අලුත් Access Token එකක් ලැබෙනවා. මෙය Seamless UX එකක්!

06 — Session-Based vs JWT
✗ Session-Based (Old)
  • Server memory/DB එකේ session එක store කරයි
  • Server 1 හි login වී → Server 2 හිදී fail වේ
  • Load balancing සඳහා Redis extra අවශ්‍ය වේ
  • Mobile apps වල cookies හරියටම වැඩ නොකරයි
  • CORS cross-origin issues පවතී
  • සෑම request එකකදීම DB session lookup එකක් කරයි
✓ JWT / Token-Based (Modern)
  • Server-side state එකක් නෑ — මෙය Stateless වේ
  • ඕනෑම Server එකකින් verify කළ හැක — නිදහසේ Scale කළ හැක
  • Extra infrastructure අවශ්‍ය නොවේ
  • React, Next.js, Mobile සඳහා ඉතා සුදුසුයි
  • CORS friendly වේ — Authorization header
  • Login වීමේදී පමණක් DB query එකක් කරයි, ඉන් පසු Signature එක පමණක් පරීක්ෂා කරයි
07 — Best Practices
01
🚫 Sensitive Data එපා
Payload එක Base64 decode කරන්න පුළුවන්. ඒ නිසා Passwords, Card numbers වැනි දේවල් JWT එකට දාන්න එපා. User ID සහ Role එක පමණක් භාවිතා කරන්න.
02
🔑 Strong Secret Key
256-bit+ random key එකක් භාවිතා කරන්න. Code එකේ Hardcode කරන්න එපා, .env file එකක් පාවිච්චි කරන්න. Production වලදී keys rotate කරන්න. application.properties හරහා @Value එකෙන් inject කරන්න.
03
⏰ Short Access Token
15 min expiry එකක් දෙන්න. Token එකක් steal කළත් සිදුවිය හැකි හානිය (damage window) කෙටියි. Refresh Token එකකින් seamlessly renew කරගන්න පුළුවන්.
04
🍪 HttpOnly Cookie
Refresh Token එක HttpOnly Secure Cookie එකක් විදියට පාවිච්චි කරන්න. JS access block වෙන නිසා XSS protection ලැබෙනවා. SameSite=Strict යෙදීමෙන් CSRF protection ලැබෙනවා.
05
📚 Library Use
io.jsonwebtoken (Spring) / jsonwebtoken (Node) වැනි libraries පාවිච්චි කරන්න. Manual JWT implementation එකකදී vulnerabilities එන්න තියෙන risk එක වැඩියි. Library එකක් use කරන එක safe.
06
🔄 Token Rotation
Refresh Token එකක් භාවිතා කළ පසු නව Refresh Token එකක් issue කරලා පරණ එක invalidate කරන්න. මෙයින් Replay attacks වළක්වන්න පුළුවන්. අවශ්‍ය නම් DB එකේ revocation list එකක් තියාගන්න.
